Simple Interactive Etch Demo

  • Info
  • Test Connection
  • Deploy Contract
  • Interact with Contract
Step 1: Make sure that you have ledger running locally. This app will support ledger networks later on.

Step 2: Go to Test Connection link and click on Test button. If you see a "successful" pop-up, you may proceed.

Step 3: Go to Deploy Contract link. Click on Choose File and select adoption.etch from etch-demos Github. The source code will automatically populate in textarea. Click Deploy, wait a few second, and you will see a "deployed" page. Once there, put http://127.0.0.1:5000/ as link to go back to home page.

Step 4: Go to Deploy Contract link. Repeat Step 3 but this time deploy hello.etch from etch-demos. Step 5: Go to Interact with Contract link. You should see the latest block info in the Ledger section. This will update everytime you refresh the "/interact" page. You will also see the all the contract names you deployed in the dropdown to the right of "Select Contract:". Each contract has an owner (type Entity), which you can find the dropdown to the to the right of "Select entity to execute function:". They will automatically update with each new deployed contract.

Step 6: To add a new non-owner Entity, type in the Entity name in the textbox in the Entity section and click the Add button. Once you go back to the "/interact" page, the new Entity name will appear in the dropdown.

Step 7: By default, in the Contract Section, the selected contract is "hello", selected action is "@query", and selected function is "persistentGreeting". If that is the case, feel free the press the Execute button without touching the [tx fee] or [arg1] textboxes. You should see a new page with "Hello World!" as that is the current result of the persistentGreeting function from the hello.etch contract you have deployed onto the ledger.

Note: This app is still WIP. For example, it does not support @action function. The next step is for the "/interact" page to to dynamically update whenever the user changes a dropdown value in the Contract Section.
